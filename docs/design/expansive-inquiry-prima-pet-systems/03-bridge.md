## Hybrid Questions

- Choreography notation
  - Analogy: Prima's sprite sheet is not just a set of pictures; it is a score for bodily transitions. A good pet system needs notation for states, transitions, timing, and expressive constraints.
  - Hybrid question: What would a pet choreography spec look like if it described motion intent rather than only frames?

- Glassblowing
  - Analogy: A glass object holds visible interior volume because its boundary is precise and translucent. Prima's internal space must be designed by controlling edges, opacity, and voids, not by simply deleting body mass.
  - Hybrid question: Can Prima's future render model treat body contour as a vessel wall around active empty space?

- Monetary policy
  - Analogy: A central bank influences behavior through signals, not direct control. A pet can shape user attention through ambient cues without interrupting the work.
  - Hybrid question: Which pet signals are strong enough to change behavior but weak enough to avoid hijacking attention?

- Fungal mycelium
  - Analogy: The visible mushroom is only the fruiting body of a wider hidden network. Prima can be the visible body of system state: tasks, tests, repo health, notifications, and workflow friction.
  - Hybrid question: What hidden substrate is Prima allowed to surface, and what must remain outside his reach?

- Stage lighting
  - Analogy: Lighting changes the audience's interpretation of the same body without changing the actor. Pet state can be communicated through color temperature, density, highlight, and flicker instead of new poses.
  - Hybrid question: Can the pet runtime separate silhouette from state lighting so identity stays stable while meaning changes?

## Most Productive Bridge

Stage lighting is the most immediately useful bridge. It suggests a clean implementation split: Prima's form stays stable, while system state is expressed through light, density, and tempo. This avoids multiplying sprite assets for every state and makes future procedural effects natural.
