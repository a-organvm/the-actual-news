## Topological Reading

Prima should be treated as a small interface body made from three simultaneous systems:

1. Visual matter
   - A lean anthropomorphic nanobot swarm with a stable acrobatic silhouette.
   - Always phasing: edges dissolve, limbs trail, internal voids open and close, particles shift between visible and absent.
   - The body should read as programmable matter, not costume, smoke, slime, or decorative sparkles.

2. Runtime state
   - The pet becomes useful when it expresses coarse workflow truth: idle, active, waiting, blocked, complete, error.
   - These states should be expressed through density, tempo, posture, flicker, and accent lighting rather than chatty narration.
   - The invariant is dynamic phasing; the variable is coherence. Blocked means less coherent. Done means briefly more resolved. Active means denser and faster.

3. Governance and distribution
   - A pet package touches install paths, metadata, spritesheets, validation, user settings, versioning, provenance, and eventually permissions.
   - Personalization belongs behind explicit boundaries. The portable pet package should not silently carry private user context.
   - If pets ever gain scripts, network access, memory, or runtime hooks, they become capability-bearing software and need signing, sandboxing, and permission declarations.

## Epistemic Signature

`material-mythic + interface-pragmatic + governance-sensitive + motion-first + state-bearing + agency-constrained`

The inquiry is mythically rich but technically restrained. The strongest shape is not "AI companion that does everything." It is "ambient state familiar whose body makes process visible."

## Productive Contradictions

- Prima should feel alive, but must not pretend to possess unearned agency.
- He should be based on the user, but should not export private user context as part of the pet package.
- He should always be unstable, but must remain readable at small UI scale.
- He should be visually quantum, but the package and runtime contract must be deterministic.
- He should invite attachment, but system truth must outrank charm.

## Three Meta-Patterns

1. Coherence under flux
   - Prima's body should be unstable at the material layer and stable at the identity layer. The silhouette carries continuity; particles carry transformation.

2. Companion as honest state surface
   - The best use of pets is not extra speech. It is peripheral legibility: a living status surface that gives the user truthful cues without interrupting the work.

3. Material myth constrained by governance
   - The more a pet feels like a familiar, the more precise the boundaries must be. Strong myth needs strong manifest discipline.

## Systems Pets Touch

- Asset pipeline
  - Sprite generation, background removal, transparency cleanup, frame extraction, contact-sheet QA, animated previews, validation.

- Package manifest
  - Name, version, species/concept, sprite dimensions, animation states, author, license, provenance, supported runtime features.

- Codex UI runtime
  - Animation state selection, scaling, frame timing, display position, accessibility fallbacks, reduced-motion behavior.

- Local user configuration
  - Active pet selection, install path, per-user preferences, optional state mappings, disabled states.

- Workflow state
  - Coarse signals such as thinking, tool-running, waiting, blocked, verified, failed, done.

- Trust and permissions
  - What the pet can observe, whether it can execute code, whether it can access memory, whether it can communicate externally.

- Distribution
  - Zip package, release artifact, checksum, changelog, preview image, install instructions, compatibility note.

- Governance
  - No secret material in packages. No false state. No hidden telemetry. No implied authority beyond actual system evidence.

## Use Model

Prima's first useful mode should be an ambient companion:

- Idle: loosely coherent, slow particle breathing, visible interior voids.
- Active: denser swarm, sharper silhouette, quicker phase transitions.
- Waiting on user: tilted attentive posture, particles held in partial suspension.
- Blocked: fragmented outline, wider voids, lower coherence.
- Complete: brief high-coherence flash, then return to breathing flux.
- Error: unstable edge shimmer, restrained warning accent, no panic animation.

The pet should not narrate these states by default. It should make them visible.

## Design Doctrine

Prima is not a mascot pasted onto Codex. Prima is a visible condensation of process: matter that has not decided whether it is body, signal, tool, or witness. The pet system should honor that by separating three concerns cleanly:

- Form: who the pet is.
- State: what the system is doing.
- Capability: what the pet is allowed to touch.

Do not collapse those into one vague "companion" concept. That collapse is where charm becomes confusion.
