## Core Inquiry

How should Prima, a lean anthropomorphic prima materia nanobot swarm that is always phasing between presence and absence, function as both a visual companion and a systems-facing interface primitive across Codex, user identity, collaboration workflows, distribution, and governance?

## Essential vs. Peripheral

Essential:

- Visual grammar: Prima must read as dynamic programmable matter, not a static mascot.
- Motion state: the pet should always feel like it is forming, unforming, and partially suspended in quantum indeterminacy.
- Use model: the pet should have a reason to exist beyond decoration, including ambient state, task context, and interaction affordances.
- Systems map: identify every layer a pet touches, from local assets and runtime animation to install, sharing, permissions, updates, and governance constraints.
- Identity boundary: Prima may reflect the user's operating style, but must not become an uncontrolled proxy for the user.
- Distribution path: shipping should preserve file integrity, metadata, provenance, versioning, and install ergonomics.

Peripheral for now:

- Exact implementation of a pet runtime API.
- Marketplace economics.
- Multi-user pet synchronization.
- Rich AI autonomy or persistent memory.
- Full animation rework beyond the current sprite asset.

## Hidden Ambiguities

- "Pet" can mean ornament, agent avatar, status display, workflow control, ritual object, or portable identity token. These are different products.
- "Always dynamic" may be represented visually by animation, procedurally by runtime effects, semantically by state changes, or socially by how it responds to context.
- "Programmable matter" can be a visual metaphor or a literal interface contract where pet behaviors are modular and scriptable.
- "Based on what you know about me" implies personalization, but personalization can drift into surveillance unless the data boundary is explicit.
- Shipping a pet can mean sharing a zip, registering it in Codex, publishing a repo artifact, releasing a plugin, or building an ecosystem standard.
